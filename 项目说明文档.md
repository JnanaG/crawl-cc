# Crawl_cc 项目说明文档

## 1. 项目定位

`Crawl_cc` 是一个面向汽车领域的智能数据工程与 RAG 项目，围绕懂车帝车系数据打通了从采集、清洗、治理、分块、建索引、检索、评测到回流优化的完整链路。

当前代码基线重点不是单一爬虫，而是一个带治理层、评测层和工作流能力的可持续演进 RAG 数据底座。

## 2. 当前能力概览

### 2.1 数据生产链

- 采集懂车帝车系详情页，保存原始 HTML 和 JSON
- 清洗出结构化 JSON 与 Markdown 文本
- 基于治理结果决定样本 `accept / repair / review / drop`
- 将通过治理的内容切分为训练 / 检索可用 chunk
- 导出 `JSONL + Parquet`

### 2.2 智能治理链

当前治理层位于 `agent_pipeline/`，主链路为：

```text
route -> rule_dedup -> semantic_dedup -> quality -> decision
```

每个节点的职责如下：

- `route`
  - 识别渠道、模板和处理策略
  - 支持规则优先、LLM 可选增强
- `rule_dedup`
  - 精确去重、规范化去重
- `semantic_dedup`
  - embedding + 向量召回做语义去重
- `quality`
  - 质量评分、问题标签、修复建议、RAG/训练 readiness
  - 支持规则打底、LLM 可选增强
- `decision`
  - 最终守门，输出 `accept / repair / review / drop`

### 2.3 RAG 与评测链

- `rag_demo.py`
  - 最小可运行的离线 RAG demo
- `rag_llm_demo.py`
  - 标准 RAG 主入口，支持 embedding / FAISS / reranker / LLM
- `rag_langchain_demo.py`
  - LangChain 风格 RAG 入口
- `rag_eval.py`
  - 传统检索评测
- `rag_ragas_eval.py`
  - `ragas` 优先、轻量 fallback 的 RAG 评测脚本

### 2.4 工作流与运维能力

- `scripts/run_pipeline_workflow.py`
  - 串联 `main -> build -> eval -> feedback -> ci_gate`
  - 支持 `dry-run`
  - 支持 `resume`
  - 支持步级重试
  - 支持失败告警与质量波动告警
- `scripts/verify_end_to_end.py`
  - 检查端到端产物是否闭合
- `scripts/sync_ragas_feedback.py`
  - 将评测低分样本回流到治理层队列

### 2.5 新增数据资产能力

- `scripts/split_processed_assets.py`
  - 把 `processed` 分流成：
    - `rag_corpus`
    - `training_corpus`
    - `eval_candidates`

## 4. 关键模块说明

### 4.1 `scraper/`

- `dcd_scraper.py`
  - 负责车系 ID 收集与车系详情抓取
  - 支持限速、重试、原始数据落盘

### 4.2 `cleaner/`

- `dcd_cleaner.py`
  - 从原始 JSON 中提取结构化字段
  - 生成适合 LLM / RAG 的 Markdown

### 4.3 `quality/`

- `data_quality.py`
  - 清洗层与训练层质量校验
  - 输出质量报告与规则诊断

### 4.4 `agent_pipeline/`

- `orchestrator.py`
  - 智能治理主入口
- `rules_bridge.py`
  - 规则路由、规则去重、规则质量诊断
- `agents.py`
  - 启发式 route / semantic_dedup / quality / decision agent
- `llm_agents.py`
  - LLM 驱动的 route / quality 增强桥接层
- `feedback_bridge.py`
  - 将评测结果转成 review / repair / feedback 队列
- `dataset_splitter.py`
  - 多目标资产分流

### 4.5 `utils/`

- `embedding_client.py`
  - 支持 `fastembed` / `sentence_transformers` / `ollama` / `openai_compatible`
- `llm_client.py`
  - 支持 `ollama` / `openai_compatible`
- `faiss_vector_store.py`
  - FAISS 向量索引读写与检索
- `llm_processor.py`
  - 分块与 token 处理
- `job_state.py`
  - 主流程断点续跑状态管理
- `stats_collector.py`
  - 报表与运行统计

## 5. 当前真实数据流

```text
懂车帝页面
  -> raw/html + raw/json
  -> cleaned/json + cleaned/markdown
  -> governance(agent pipeline)
  -> processed/jsonl + parquet
  -> assets split
  -> vector store
  -> rag query / rag eval
  -> feedback / review / repair
```

## 6. LLM 接入现状

### 6.1 已接入 LLM 的节点

- `route`
- `quality`

这两个节点适合 LLM 的原因：

- `route` 需要做弱结构化语义判断
- `quality` 需要做问题归因和修复建议增强

### 6.2 未接入 LLM 的节点

- `rule_dedup`
- `semantic_dedup`
- `decision`

原因：

- `rule_dedup` 是确定性哈希问题，不需要 LLM
- `semantic_dedup` 更适合 embedding + 阈值判断
- `decision` 是最终守门器，当前仍保留确定性规则控制

### 6.3 运行方式

单条样本关闭 LLM：

```powershell
python scripts/run_governance_without_llm.py --semantic-store-mode stub
```

单条样本开启 LLM：

```powershell
python scripts/run_governance_with_llm.py ^
  --semantic-store-mode stub ^
  --llm-provider ollama ^
  --llm-model qwen2.5:3b ^
  --llm-api-base http://localhost:11434
```

主流程开启 LLM：

```powershell
$env:AGENT_ENABLE_LLM="true"
$env:AGENT_LLM_PROVIDER="ollama"
$env:AGENT_LLM_MODEL="qwen2.5:3b"
$env:AGENT_LLM_API_BASE="http://localhost:11434"
python main.py
```

## 7. 当前推荐运行方式

### 7.1 主流程

```powershell
python main.py
```

### 7.2 资产分流

```powershell
python scripts/split_processed_assets.py
```

### 7.3 向量构建

```powershell
python rag_llm_demo.py build --input data/assets/rag/rag_corpus.jsonl --embedding-provider fastembed
```

### 7.4 RAG 评测

```powershell
python rag_ragas_eval.py ^
  --golden-set data/evaluation/golden_set.jsonl ^
  --output data/evaluation/ragas_results.json ^
  --embedding-provider fastembed ^
  --llm-provider ollama ^
  --llm-model qwen2.5:3b
```

### 7.5 回流与守门

```powershell
python scripts/sync_ragas_feedback.py --results data/evaluation/ragas_results.json
python scripts/ci_eval_gate.py --results data/evaluation/ragas_results.json
```

### 7.6 一键工作流

```powershell
python scripts/run_pipeline_workflow.py --run-ci-gate --llm-provider ollama --llm-model qwen2.5:3b
```

## 9. 当前 Git 上传策略

`.gitignore` 默认排除：

- `.venv/`
- `venv/`
- `env/`
- `data/`
- 日志文件
- 本地 IDE 配置

这意味着：

- 代码、文档、脚本会进 Git
- 运行产物、数据文件、虚拟环境不会上传
