## 1. 适用范围

本文档对应当前仓库中已经落地的新爬取策略，核心入口在 [refetch_series_modules.py](file:///D:/workplace/Crawl_cc/scripts/refetch_series_modules.py)、[dcd_scraper.py](file:///D:/workplace/Crawl_cc/scraper/dcd_scraper.py)、[dcd_cleaner.py](file:///D:/workplace/Crawl_cc/cleaner/dcd_cleaner.py)。

当前抓取范围固定为 3 个模块：

- `https://www.dongchedi.com/auto/series/<series_id>`
  - 只保留基础信息、价格、评分、车型列表等
  - 不保留新闻
- `https://www.dongchedi.com/auto/series/<series_id>/images-wg`
  - 外观模块
  - 每个车型选 1 张首图样本
  - 语义要求是“正面前脸图”
- `https://www.dongchedi.com/auto/series/<series_id>/images-ns`
  - 内饰模块
  - 每个车型选 1 张首图样本
  - 语义要求是“正面主副驾拍摄图”

说明：

- 当前“正面前脸图 / 正面主副驾图”的实现方式，是基于懂车帝模块页返回顺序，默认取 `pic_url[0]`
- 这是一种工程上的稳定约束，不是视觉模型判断

## 4. 两种常用爬取方式

### 4.1 在当前已有数据上继续补抓新数据

适用场景：

- 你已经有一批 `raw/cleaned` 数据
- 现在想继续补抓新的 `series_id`
- 不希望清空旧数据

命令示例：

```bash
python scripts/refetch_series_modules.py --series-ids 25634,99,575 --target-count 3
```

说明：

- 不加 `--clear-data`
- 脚本会直接对给定的 `series_id` 执行三模块抓取
- 如果某个 `series_id` 已经存在，会被新的抓取结果覆盖
- 其他未指定的旧数据会保留

如果你想在现有数据基础上自动再补一批：

```bash
python scripts/refetch_series_modules.py --target-count 30 --max-expand-requests 20
```

说明：

- 这会自动从首页、公共入口页、搜索页收集车系池
- 然后补抓 `30` 个车系
- 旧数据不会被清空

### 4.2 从头清空后按新策略全量重爬

适用场景：

- 你要彻底切换到新三模块方案
- 不想保留旧版 raw/cleaned/processed/multimodal/assets 产物

命令示例：

```bash
python scripts/refetch_series_modules.py --clear-data --target-count 50 --max-expand-requests 30
```

说明：

- `--clear-data` 会清理以下目录和状态：
- `data/raw/dongchedi`
- `data/cleaned/dongchedi`
- `data/processed`
- `data/multimodal`
- `data/assets`
- `data/state/agent_pipeline`
- `data/state/job_state.json`

重爬汇总文件默认输出到：

- [refetch_series_summary.json](file:///D:/workplace/Crawl_cc/data/reports/refetch_series_summary.json)

## 5. 爬取后的推荐全流程

这里分成两条主线说明。

### 5.1 主线 A：新三模块多模态 RAG 流程

这是当前最推荐的流程，适合你现在的项目方向。

#### 第一步：执行三模块爬取

全量重建：

```bash
python scripts/refetch_series_modules.py --clear-data --target-count 50 --max-expand-requests 30
```

增量补抓：

```bash
python scripts/refetch_series_modules.py --series-ids 25634,6325,4363 --target-count 3
```

#### 第二步：构建图片资产

```bash
python scripts/build_image_dataset.py
```

对应脚本：[build_image_dataset.py](file:///D:/workplace/Crawl_cc/scripts/build_image_dataset.py)

输出：

- `data/multimodal/image_assets.jsonl`
- `data/multimodal/image_classification_manifest.jsonl`
- `data/multimodal/image_dataset_summary.json`

#### 第三步：生成图片 caption 语料

先用本地离线可跑版本：

```bash
python scripts/build_image_caption_corpus.py --caption-provider heuristic
```

如果接真实多模态模型：

```bash
python scripts/build_image_caption_corpus.py ^
  --caption-provider openai_compatible ^
  --caption-model your-vl-model ^
  --caption-api-base http://localhost:8000/v1 ^
  --caption-api-key your_key
```

对应脚本：[build_image_caption_corpus.py](file:///D:/workplace/Crawl_cc/scripts/build_image_caption_corpus.py)

输出：

- `data/multimodal/image_caption_corpus.jsonl`
- `data/multimodal/image_caption_generation_log.jsonl`
- `data/multimodal/image_caption_summary.json`

#### 第四步：构建向量库

如果你已经有文本 `processed` 数据：

```bash
python rag_llm_demo.py build ^
  --input data/processed/dongchedi_training_data.jsonl ^
  --extra-inputs data/multimodal/image_caption_corpus.jsonl ^
  --embedding-provider hash
```

如果当前只先验证图片 caption RAG，也可以先只拿图片 caption 建库：

```bash
python rag_llm_demo.py build ^
  --input data/multimodal/image_caption_corpus.jsonl ^
  --embedding-provider hash
```

对应入口：[rag_llm_demo.py](file:///D:/workplace/Crawl_cc/rag_llm_demo.py)

输出目录：

- `data/vector_store/faiss/`

#### 第五步：启动多轮 API

如果用 FAISS 后端：

```bash
python scripts/serve_multi_turn_api.py --backend-mode faiss --host 127.0.0.1 --port 8001
```

如果先本地快速验证：

```bash
python scripts/serve_multi_turn_api.py --backend-mode hash --host 127.0.0.1 --port 8001
```

对应脚本：[serve_multi_turn_api.py](file:///D:/workplace/Crawl_cc/scripts/serve_multi_turn_api.py)

### 5.2 主线 B：标准文本 processed 全流程

这是仓库里原本较完整的“抓取 -> 治理 -> chunk -> processed -> 报告”流水线，入口是 [main.py](file:///D:/workplace/Crawl_cc/main.py)。

#### 推荐使用方式

如果你要完整生成：

- `data/processed/dongchedi_training_data.jsonl`
- `data/processed/dongchedi_training_data.parquet`
- `data/reports/*`

建议直接从清洁状态跑：

```bash
python main.py
```

可选环境变量：

```bash
set TARGET_TRAINING_RECORDS=1000
set TARGET_SERIES_POOL=300
set MAX_CHUNK_TOKENS=100
set MAX_SERIES_EXPAND_REQUESTS=40
python main.py
```

说明：

- `main.py` 当前内部也会调用新的 [DongchediScraper](file:///D:/workplace/Crawl_cc/scraper/dcd_scraper.py)
- 所以它抓到的已经是“主页 + images-wg + images-ns”的新结构
- 但它仍然是“一体化流水线”，会重新抓、重新清洗、重新治理和导出 processed

#### 跑完后的产物

- `data/processed/dongchedi_training_data.jsonl`
- `data/processed/dongchedi_training_data.parquet`
- `data/cleaned/dongchedi/cleaned_series.parquet`
- `data/reports/quality_report.json`
- `data/reports/quality_report.md`
- `data/reports/task_summary.json`

#### 继续做资产分流

```bash
python scripts/split_processed_assets.py
```

对应脚本：[split_processed_assets.py](file:///D:/workplace/Crawl_cc/scripts/split_processed_assets.py)

输出：

- `data/assets/rag/rag_corpus.jsonl`
- `data/assets/training/training_corpus.jsonl`
- `data/assets/evaluation/eval_candidates.jsonl`

## 6. 评估流程

### 6.1 先有向量库

```bash
python rag_llm_demo.py build ^
  --input data/processed/dongchedi_training_data.jsonl ^
  --extra-inputs data/multimodal/image_caption_corpus.jsonl ^
  --embedding-provider hash
```

### 6.2 再执行评估

```bash
python rag_ragas_eval.py ^
  --golden-set data/evaluation/golden_set.jsonl ^
  --index data/vector_store/faiss/dongchedi.index ^
  --records data/vector_store/faiss/dongchedi_records.jsonl ^
  --meta data/vector_store/faiss/dongchedi_meta.json
```

对应脚本：[rag_ragas_eval.py](file:///D:/workplace/Crawl_cc/rag_ragas_eval.py)

说明：

- 该脚本优先尝试真实 `ragas`
- 如果当前环境依赖不兼容，会自动退回 lightweight 评估逻辑
